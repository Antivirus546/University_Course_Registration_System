Abstract
The University Course Registration System is a relational database project designed
to digitize and streamline the academic enrollment process. Manual course registration
is often prone to errors, data redundancy, and inefficiencies in tracking seat availability.
This project aims to resolve these issues by implementing a robust database backend in-
terfaced with a user-friendly GUI.

Project Scope and Functionality: The system focuses on managing the many-to-many
relationship between students and courses. Key features include:

Student Management: Storing student details and academic history.
Course Catalog: Maintaining a list of offered courses, including credits, instructor
details, and prerequisites.
Registration Logic: allowing students to enroll in courses while the system auto-
matically checks for constraints such as prerequisite completion, schedule clashes,
and seat limits.
Implementation Details: The project will be implemented using MySQL as the backend
database management system. We will utilize SQL DDL commands to define the schema
with appropriate Primary and Foreign Keys to ensure referential integrity. The frontend
will be a GUI application (using Python/Java) that connects to the database to execute SQL
queries for inserting, updating, and retrieving registration data.
